# 设计文档

## 1. 项目目标

本项目用于学习 SDL 与 FFmpeg 在桌面播放器中的协作方式，优先关注以下主题：

- 媒体打开与流探测
- 音视频解码与像素/采样格式转换
- 音视频同步
- seek
- 简洁 UI 控制层
- 面向学习的清晰注释与模块边界

## 2. 为什么首版选择 SDL2

需求里希望尽量使用较新的稳定版本。实际落地时，首版代码选择：

- `SDL2 2.30+`
- `FFmpeg 7.x`

原因：

1. SDL2 在 Windows + MSVC 场景资料更成熟。
2. 音频回调、渲染和事件系统 API 更稳定，适合作为学习版播放器起点。
3. 等核心架构稳定后，可以较平滑迁移到 SDL3。

## 3. 总体架构

播放器主要分为三层：

### 3.1 UI 层

位于 `src/main.cpp`，负责：

- SDL 窗口创建
- 事件循环
- 进度条与按钮交互
- 基础文字信息显示

### 3.2 Player 控制层

位于 `src/player/Player.h/.cpp`，负责：

- 打开/关闭媒体
- 播放状态管理
- seek 请求
- 倍速切换
- 当前进度与媒体信息查询
- 视频帧渲染输出

### 3.3 媒体处理层

当前在 `Player.cpp` 内部实现，后续可继续拆分：

- `libavformat`：打开文件、读取 packet
- `libavcodec`：音频 / 视频解码
- `libswresample`：音频重采样
- `libswscale`：视频像素格式转换
- SDL audio callback：PCM 输出

## 4. 线程模型

当前版本采用“主线程 + 一个媒体工作线程 + SDL 音频回调”的结构：

### 4.1 主线程

- 处理 SDL 事件
- 绘制 UI
- 依据时钟刷新当前显示帧

### 4.2 媒体工作线程

- 持续 `av_read_frame`
- 将 packet 送入音频 / 视频解码器
- 将音频 frame 转成 PCM chunk
- 将视频 frame 转成 RGBA frame
- 把结果压入各自队列

### 4.3 SDL 音频回调

- 从音频 chunk 队列读取 PCM
- 将数据写入 SDL 输出缓冲
- 同步更新“当前正在播放的媒体时间”

这种设计比 ffplay 的完整多线程模型简单，更适合作为学习版本，但已经能体现播放器核心链路。

## 5. 音视频同步策略

### 5.1 音频作为主时钟

若媒体包含音频，则使用音频播放进度作为主时钟：

- 每个音频 chunk 都记录其对应的媒体时间区间
- 音频回调每消费一部分 PCM，就能换算出当前实际播放到的媒体时间
- 视频渲染时选择最接近当前音频时钟的帧显示

### 5.2 无音频时的视频时钟

若媒体没有音频，则使用：

- `anchorPosition`
- `anchorWallClock`
- `speed`

推导当前视频播放时间，保证纯视频或无音频时仍可推进时间轴。

## 6. Seek 设计

当用户拖动时间轴时：

1. 主线程只提交 seek 请求
2. 媒体工作线程执行 `av_seek_frame`
3. 刷新解码器状态
4. 清空音频 / 视频队列
5. 重置时钟锚点

这样做的关键点是：seek 必须在统一线程中完成，避免读包、解码与状态刷新交错导致脏状态。

## 7. 倍速设计

当前倍速采用“重采样输出速率变化”的学习型实现：

- 保持 SDL 音频设备输出采样率固定
- 调整 `libswresample` 的输出采样率，生成不同长度的 PCM
- 这样可以实现变速播放

副作用：

- 这是“变速且变调”的方案
- 优点是实现简单，便于理解播放器链路
- 若要做更高质量的变速不变调，需要引入额外的 time-stretch 算法或 DSP 处理

## 8. UI 设计

当前 UI 采用 SDL 自绘方案，包含：

- 主内容区：视频画面或仅音频占位区
- 底部控制栏：播放、打开文件、倍速按钮
- 进度条：支持鼠标拖动
- 信息区：显示文件名、容器、编码、分辨率、采样率等

设计原则：

- 结构清晰，便于继续扩展
- 控件不追求复杂样式，优先服务学习

## 9. 支持范围说明

实际可支持的容器/编码依赖 FFmpeg：

- 容器：`mp4, mkv, mov, avi, flv, mp3, aac, wav`
- 若 FFmpeg 启用了相关协议和 demux，后续可扩展 `m3u8/HLS`
- 视频编码：优先覆盖 `H.264/AVC`，在 FFmpeg 支持时可解码 `H.265/HEVC`
- 音频编码：AAC、MP3、PCM 等常见格式

## 10. 后续演进建议

可继续拆分出以下模块：

- `Demuxer`
- `AudioRenderer`
- `VideoRenderer`
- `Clock`
- `SeekController`
- `OverlayUi`

等项目从学习版走向更完整的工程后，再逐步把当前 `Player.cpp` 中的逻辑拆散，会比一开始就过度抽象更稳妥。
