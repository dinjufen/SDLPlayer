# 格式与编码支持矩阵

## 说明

本项目的容器与编码支持能力，最终由运行时 FFmpeg 的编译配置决定。

当前代码结构已经围绕 `libavformat + libavcodec + libswresample + libswscale` 搭好通用解复用/解码链路，因此在 FFmpeg 具备对应 demuxer/decoder 时，播放器具备接入这些格式的基础能力。

## 首版目标

首版重点是：

- 本地文件播放
- 音频与视频统一播放器状态机
- 纯音频播放路径
- 进度条、seek、倍速与媒体信息展示

`m3u8 / HLS` 已在需求中保留，但属于下一阶段增强目标。

## 编码支持规划

| 类型 | 编码 | 当前规划 |
| --- | --- | --- |
| 视频 | `H.264 / AVC` | 首版重点验证 |
| 视频 | `H.265 / HEVC` | 在 FFmpeg 支持时优先纳入 |
| 音频 | `AAC` | 首版重点验证 |
| 音频 | `MP3` | 首版重点验证 |
| 音频 | `PCM / WAV` | 首版重点验证 |

## 当前代码已经覆盖的通用能力

- 自动选择最佳音频流和视频流
- 视频帧解码与 BGRA 转换
- 音频帧解码与 S16 双声道重采样
- 音频主时钟驱动的视频同步
- 纯音频文件的无视频占位模式
- 基础媒体信息展示

这些能力对多数主流封装格式是通用的，不依赖单一格式写死逻辑。

## 当前会话中的验证边界

在本会话环境中，已经验证：

- 工程目录结构、CMake 配置、preset 定义与 README 命令说明
- 本机存在可被 CMake 识别的 MSVC 编译器环境

- 因而尚未完成 CTest 执行确认，以及样片播放、seek、EOF 与音视频同步的端到端回归

## 建议测试素材

后续完成依赖安装后，建议至少准备以下样片：

- `mp4 + H.264 + AAC`
- `mkv + H.264 + AAC`
- `mov + H.264 + AAC`
- `avi + MPEG4/H.264 + MP3`
- `flv + H.264 + AAC`
- `mp3`
- `aac`
- `wav`
- `mp4 / mkv + H.265 + AAC`（若 FFmpeg 构建包含 HEVC 解码）

## 建议验证项

每类素材建议验证：

1. 能否正常打开
2. 时长是否正确显示
3. 播放/暂停是否正常
4. 进度拖动是否正常
5. 倍速是否正常
6. 纯音频文件是否进入仅音频界面
7. 媒体信息是否正确显示
